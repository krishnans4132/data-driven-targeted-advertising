\justifying

This project focuses on analyzing advertisement fatigue in mobile applications using business analytics techniques. Mobile applications widely rely on advertisements such as banner ads, interstitial ads, and video ads to generate revenue. However, excessive advertising can negatively affect user experience and may lead to frustration, reduced engagement, or application uninstallation.

The study combines primary survey data and secondary review data to analyze user perceptions toward advertisements in mobile applications. The primary dataset was collected using a structured Google Forms survey capturing user demographics, mobile usage patterns, advertisement exposure frequency, and behavioral reactions toward advertisements. In addition, secondary data consisting of Google Play Store reviews was collected through automated web scraping and processed using text mining and sentiment analysis techniques.

Data preprocessing and exploratory data analysis (EDA) were performed to identify patterns in user behavior and advertisement exposure. Natural Language Processing (NLP) techniques were applied to analyze review sentiments and detect advertisement-related keywords. An Advertisement Fatigue Index was computed from Likert-scale survey responses to quantify the degree of advertisement fatigue experienced by individual users.

Advanced analytics techniques including machine learning regression models (Linear Regression, Decision Tree, and Random Forest), K-Means clustering for user segmentation, and Prophet-based time-series forecasting of advertisement complaint trends were applied to derive deeper insights. An interactive dashboard was developed to communicate findings to business stakeholders.

The findings provide valuable insights into how advertisement strategies influence user satisfaction and application engagement. These insights can help mobile app developers and digital marketers design more effective advertisement strategies that balance revenue generation with user experience.

\section{Introduction}
% ======================================================

Mobile applications have become an essential part of everyday life, providing services such as communication, entertainment, education, and shopping. Many mobile applications rely on digital advertising as a primary source of revenue. Advertisement formats such as banner ads, interstitial ads, and rewarded video ads allow companies to offer free services while generating income through marketing partnerships.

However, the increasing frequency of advertisements in mobile applications has raised concerns about advertisement fatigue among users. Advertisement fatigue occurs when users are repeatedly exposed to advertisements, leading to annoyance, decreased engagement, and reduced satisfaction with the application. When advertisements interrupt user activities or appear too frequently, users may close the application or uninstall it entirely.

Understanding how users react to advertisements is therefore an important problem for businesses and developers. By analyzing user behavior and feedback, organizations can identify how advertisement exposure affects user experience and determine which advertisement formats are acceptable to users.

This project applies a comprehensive set of business analytics techniques to study advertisement fatigue in mobile applications. By analyzing survey data, application review sentiments, predictive models, and time-series forecasts, the study aims to identify patterns in advertisement exposure, user frustration levels, and behavioral responses to advertisements.

The insights derived from this analysis can help mobile application developers design better advertising strategies that improve user experience while maintaining sustainable revenue generation.

\section{Dataset Selection and Understanding}
% ======================================================

The datasets used in this project consist of both primary and secondary data sources.

\subsection{Primary Dataset — Survey Data}

The primary dataset was collected using a structured Google Forms survey designed to understand user experiences with advertisements in mobile applications. The survey was designed with closed-ended questions to enable quantitative analysis. A total of 422 responses were collected out of a target of 1000.

The survey captures the following variables:

\begin{itemize}
    \item \textbf{Demographics}: Age group, gender, occupation, smartphone type.
    \item \textbf{Usage Patterns}: Daily screen time, daily app categories used, most used app category.
    \item \textbf{Advertisement Exposure}: Ad frequency, frequent ad format, most excessive ad category.
    \item \textbf{User Reactions}: Whether ads interrupt usage, cause frustration, reduce enjoyment, cause app closure, lead to app uninstallation, and willingness to pay for ad-free versions.
    \item \textbf{Preferences}: Most tolerated ad format, most annoying ad timing.
\end{itemize}

\subsection{Secondary Dataset — Google Play Store Reviews}

The secondary dataset was created by collecting user reviews from the Google Play Store using the \texttt{google-play-scraper} Python library. Reviews were collected across multiple app categories and countries. The raw dataset (\texttt{playstore\_reviews\_raw.csv}) was processed through the following NLP pipeline:

\begin{enumerate}
    \item \textbf{Web Scraping}: Automated extraction of reviews using \texttt{google-play-scraper}.
    \item \textbf{Text Cleaning}: Removal of punctuation, stopwords, URLs, and special characters; lowercasing and normalization.
    \item \textbf{Keyword Detection}: Classification of each review for the presence of advertisement-related keywords from a curated vocabulary list.
    \item \textbf{Sentiment Analysis}: Classification of each review as positive, neutral, or negative using TextBlob polarity scores.
\end{enumerate}

The processed dataset (\texttt{reviews\_cleaned.csv}) contains the following key fields:

\section{Data Preprocessing}
% ======================================================

\subsection{Review Data Preprocessing (Notebook 02)}

The raw review dataset (\texttt{playstore\_reviews\_raw.csv}) underwent the following cleaning steps:

\begin{itemize}
    \item Removal of null and empty review entries.
    \item Lowercasing of all review text.
    \item Removal of URLs, special characters, and punctuation using regular expressions.
    \item Removal of English stopwords to reduce noise.
    \item Stripping of extra whitespace.
    \item Retention of only alphabetic tokens meaningful for analysis.
\end{itemize}

The cleaned dataset (\texttt{reviews\_cleaned.csv}) was saved and used as the input for text mining and sentiment analysis.

\subsection{Survey Data Preprocessing (Notebook 05)}

The raw survey dataset (\texttt{survey\_dataset.csv}) was preprocessed through the following steps:

\begin{itemize}
    \item Column renaming to snake\_case variable names for consistency: \texttt{age\_group}, \texttt{occupation}, \texttt{screen\_time}, \texttt{ad\_frequency}, \texttt{ads\_interrupt\_usage}, \texttt{ads\_cause\_frustration}, \texttt{ads\_reduce\_enjoyment}, \texttt{ads\_close\_app}, \texttt{ads\_uninstall\_app}, \texttt{pay\_for\_adfree}, \texttt{tolerated\_ad\_format}, \texttt{most\_annoying\_ad\_time}, and others.
    \item Removal of the timestamp column (not analytically relevant).
    \item Missing value imputation using mode values for categorical columns.
    \item Deduplication of repeated survey responses.
    \item Type conversion for consistent data types across columns.
\end{itemize}

\subsection{Ad Fatigue Index Computation}

An Advertisement Fatigue Index (AFI) was engineered as a composite score from four Likert-scale survey questions measuring the degree to which advertisements affect user experience. The mapping applied was:

\[
\text{Score} = \begin{cases}
1 & \text{Strongly Disagree} \\
2 & \text{Disagree} \\
3 & \text{Neutral} \\
4 & \text{Agree} \\
5 & \text{Strongly Agree}
\end{cases}
\]

The four component columns were:
\begin{itemize}
    \item \texttt{ads\_interrupt\_usage\_score}
    \item \texttt{ads\_cause\_frustration\_score}
    \item \texttt{ads\_reduce\_enjoyment\_score}
    \item \texttt{ads\_close\_app\_score}
\end{itemize}

The Ad Fatigue Index was computed as the mean of these four scores per respondent:

\[
\text{Ad Fatigue Index} = \frac{\text{interrupt} + \text{frustration} + \text{enjoyment} + \text{close}}{4}
\]

This index ranges from 1 (no fatigue) to 5 (extreme fatigue) and serves as the primary dependent variable for regression and clustering analyses.

\section{Sentiment Analysis and Text Mining}
% ======================================================

\subsection{Advertisement Keyword Detection (Notebook 03)}

A curated vocabulary of advertisement-related keywords was constructed. This list encompasses terms such as: \textit{ads}, \textit{advertisement}, \textit{too many ads}, \textit{ads everywhere}, \textit{video ads}, \textit{pop-up ads}, \textit{banner ads}, \textit{invasive ads}, \textit{forced ads}, and further compound phrases describing frustration with advertisements.

For each review, a binary flag (\texttt{ad\_mentioned}) was assigned indicating whether any keyword from the vocabulary was present in the cleaned review text. The first matched keyword was stored in the \texttt{ad\_keyword} field.

\subsection{Sentiment Classification}

Sentiment was classified using the TextBlob library's polarity scoring:

\[
\text{Sentiment} = \begin{cases}
\text{Positive} & \text{if polarity} > 0 \\
\text{Negative} & \text{if polarity} < 0 \\
\text{Neutral} & \text{if polarity} = 0
\end{cases}
\]

This classification enables the analysis of review tone, particularly for reviews containing ad-related complaints.

\section{Machine Learning — Ad Fatigue Prediction}
% ======================================================

\subsection{Feature Engineering and Encoding (\texttt{08\_Ad Fatigue Prediction.ipynb})}

To prepare the survey dataset for machine learning, the following preprocessing steps were applied:

\begin{itemize}
    \item Label encoding of all categorical columns using \texttt{scikit-learn}'s \texttt{LabelEncoder}.
    \item Median imputation for any remaining missing numerical values.
    \item Correlation matrix computation to identify relationships between features and the target variable.
\end{itemize}

\subsection{Regression Models}

The target variable for regression was the \textbf{Ad Fatigue Index}. The dataset was split into training (80\%) and test (20\%) sets. Three regression models were trained and evaluated:

\begin{enumerate}
    \item \textbf{Linear Regression} --- A baseline model assuming a linear relationship between features and the Ad Fatigue Index.
    \item \textbf{Decision Tree Regressor} --- A non-linear tree-based model that recursively partitions the feature space.
    \item \textbf{Random Forest Regressor} --- An ensemble of 200 decision trees with random feature subsampling, providing improved generalization.
\end{enumerate}

Model performance was evaluated using three metrics:

\[
\text{MAE} = \frac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|
\]
\[
\text{RMSE} = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2}
\]
\[
R^2 = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}
\]

\begin{table}[H]
\centering
\begin{tabular}{lccc}
\toprule
\textbf{Model} & \textbf{MAE} & \textbf{RMSE} & \textbf{R\textsuperscript{2}} \\
\midrule
Linear Regression & 0.815 & 0.994 & 0.075 \\
Decision Tree Regressor & 0.404 & 0.565 & 0.701 \\
Random Forest Regressor & 0.282 & 0.358 & 0.880 \\
\bottomrule
\end{tabular}
\caption{Regression Model Comparison}
\end{table}

\subsection{Feature Importance}

The Random Forest model's feature importances were extracted and visualized to identify the top predictors of advertisement fatigue. The top 10 most influential features were plotted using a bar chart, providing interpretability of the model and actionable insights for advertisement strategy.

\subsection{Model Interpretation}

The regression results indicate that tree-based models perform significantly better than the linear baseline model for predicting the Advertisement Fatigue Index. The Linear Regression model achieved a relatively low R\textsuperscript{2} value, suggesting that the relationship between advertisement exposure variables and fatigue levels is not purely linear.

The Decision Tree Regressor improved the prediction accuracy by capturing non-linear interactions between features such as advertisement frequency, ad format, and user behavior. However, single decision trees may suffer from overfitting.

The Random Forest Regressor produced the best performance among the evaluated models, achieving the lowest MAE and RMSE values and the highest R\textsuperscript{2} score. This improvement is due to the ensemble nature of Random Forest, which aggregates predictions from multiple trees and reduces variance.

These results suggest that advertisement fatigue is influenced by complex interactions among multiple factors, and ensemble learning methods are more suitable for capturing these patterns.

\section{User Segmentation — K-Means Clustering (\texttt{09\_k-clustering.ipynb})}
% ======================================================

A dedicated clustering notebook was developed to segment survey respondents into distinct behavioral groups based on their advertisement reaction patterns. This notebook uses a richer feature set than the preliminary clustering in the prediction notebook and applies PCA for visualization.

\subsection{Feature Selection and Preprocessing}

Five features were selected from the cleaned survey dataset (\texttt{survey\_cleaned.csv}) for clustering:

\begin{itemize}
    \item \texttt{ads\_interrupt\_usage\_score}
    \item \texttt{ads\_cause\_frustration\_score}
    \item \texttt{ads\_reduce\_enjoyment\_score}
    \item \texttt{ads\_close\_app\_score}
    \item \texttt{ad\_fatigue\_index} (composite score)
\end{itemize}

Missing values were imputed using column means. All features were then standardized using \texttt{StandardScaler} to ensure equal weighting across variables with different scales.

\subsection{Optimal Cluster Selection — Elbow Method}

The optimal number of clusters was determined using the Elbow Method. Within-Cluster Sum of Squares (WCSS) was computed for $k = 1$ to $k = 10$:

\[
\text{WCSS} = \sum_{k} \sum_{x_i \in C_k} \|x_i - \mu_k\|^2
\]

The resulting elbow plot showed a clear inflection point at $k = 3$, confirming that three clusters optimally capture the variation in user advertisement behavior.

\subsection{K-Means Clustering — Full Feature Set}

K-Means clustering ($k = 3$, \texttt{random\_state=42}) was applied to the full 5-feature standardized dataset. Cluster labels were assigned to each respondent in the survey dataframe.

Principal Component Analysis (PCA) was applied to reduce the 5-dimensional feature space to 2 principal components for visualization. The cluster centroids were also projected into PCA space and overlaid as markers (black `X') on the scatter plot.

\textbf{Plot: User Clusters based on Advertisement Behaviour}
\begin{itemize}
    \item X-axis: Principal Component 1
    \item Y-axis: Principal Component 2
    \item Colour: Cluster assignment (viridis palette)
    \item Markers: Cluster centroids (black X)
\end{itemize}

This visualization reveals three well-separated user segments:
\begin{itemize}
    \item \textbf{Cluster 0 — High Tolerance Users}: Low frustration scores and low Ad Fatigue Index. These users are generally unaffected by advertisements.
    \item \textbf{Cluster 1 — Moderate Fatigue Users}: Moderate scores across all features. These users experience occasional advertisement disruption but have mixed reactions.
    \item \textbf{Cluster 2 — High Fatigue Users}: High frustration and interruption scores combined with a high Ad Fatigue Index. These users are most at risk of closing or uninstalling applications.
\end{itemize}

\subsection{K-Means Clustering — Reduced Feature Set}

A second clustering analysis was performed using a reduced subset of three features:

\begin{itemize}
    \item \texttt{ads\_interrupt\_usage\_score}
    \item \texttt{ads\_close\_app\_score}
    \item \texttt{ad\_fatigue\_index}
\end{itemize}

This reduced model captures the most direct behavioral outcomes of advertisement exposure. PCA was again applied for 2D visualization, with centroids projected and overlaid.

\textbf{Plot: Clusters based on Advertisement Fatigue}
\begin{itemize}
    \item X-axis: Principal Component 1
    \item Y-axis: Principal Component 2
    \item Colour: Cluster assignment (plasma palette)
    \item Markers: Cluster centroids (black X)
\end{itemize}

The reduced-feature clustering corroborates the findings of the full-feature model, confirming the robustness of the three-segment user structure.

\section{Time-Series Forecasting of Advertisement Complaints}
% ======================================================

\subsection{Dataset Preparation (\texttt{07\_Time-Series Forecasting.ipynb})}

To enable time-series analysis, the secondary review dataset was aggregated by review date to compute:
\begin{itemize}
    \item \textbf{Daily ad mention count} (\texttt{y}): Total number of reviews per day flagged as containing advertisement keywords.
    \item \textbf{Daily average sentiment score}: Encoded as $+1$ (positive), $0$ (neutral), $-1$ (negative).
\end{itemize}

The daily time series was indexed over the complete date range (gaps filled with zero), and split into a training set and a 30-day hold-out test set.

\subsection{Prophet Forecasting Model}

Facebook's Prophet model was used for forecasting due to its ability to handle daily and weekly seasonality patterns and missing data. The model was configured with:

\begin{itemize}
    \item \textbf{Daily seasonality}: Enabled.
    \item \textbf{Weekly seasonality}: Enabled.
    \item \textbf{Yearly seasonality}: Disabled (insufficient data span).
\end{itemize}

An extended model was also trained with average daily sentiment score added as an external regressor, capturing the relationship between overall user sentiment and advertisement complaint volume.

\subsection{Forecast Evaluation}

The forecast was evaluated over the 30-day test period using MAE and RMSE:

\begin{table}[H]
\centering
\begin{tabular}{lc}
\toprule
\textbf{Metric} & \textbf{Value} \\
\midrule
Mean Absolute Error (MAE) & 67.62 \\
Root Mean Squared Error (RMSE) & 79.94 \\
Forecasted Ad Mentions (Next 30 Days) & 1845 \\
\bottomrule
\end{tabular}
\caption{Prophet Forecast Evaluation Metrics}
\end{table}

The forecast visualization and component decomposition plots (trend, weekly seasonality) were generated using Prophet's built-in plotting utilities.

\subsection{Forecast Interpretation}

The forecasting results indicate that advertisement-related complaints exhibit noticeable temporal patterns. The Prophet model captured short-term fluctuations and underlying trends in the daily volume of advertisement mentions in user reviews.

The Mean Absolute Error (MAE) and Root Mean Squared Error (RMSE) values demonstrate that the model is capable of reasonably approximating the daily complaint counts within the evaluation window. While some variability exists due to irregular spikes in review activity, the forecast remains useful for identifying overall complaint trends.

The model predicts approximately 1845 advertisement-related mentions over the next 30 days. This projection suggests that advertisement-related dissatisfaction is likely to persist in the near future if current advertising strategies remain unchanged.

These results highlight the importance of monitoring user feedback trends continuously. Forecasting tools such as Prophet allow organizations to proactively adjust advertisement strategies before complaint levels significantly impact user ratings and application retention.

\section{Key Findings and Insights}
% ======================================================

Based on the analyses performed, the following key insights were derived:

\subsection{From Review Sentiment Analysis}
\begin{itemize}
    \item The majority of advertisement-related reviews carry a negative sentiment, indicating widespread user dissatisfaction with in-app advertising.
    \item Game and Entertainment app categories have the highest frequency of advertisement complaints among all app categories.
    \item Reviews mentioning advertisements are significantly associated with lower star ratings compared to reviews without ad complaints.
    \item The most common advertisement complaint keywords include ``too many ads,'' ``ads everywhere,'' and ``forced ads.''
\end{itemize}

\subsection{From Survey Analysis}
\begin{itemize}
    \item The majority of surveyed users report that advertisements interrupt their mobile application usage at least occasionally.
    \item A significant proportion of respondents have uninstalled a mobile application due to excessive advertisements.
    \item Rewarded video advertisements are the most tolerated format, while interstitial (full-screen) ads are the most disruptive.
    \item Users with higher daily screen time tend to display higher Ad Fatigue Index scores.
    \item Younger age groups (18--24) report higher advertisement fatigue compared to older demographics.
\end{itemize}

\subsection{From Machine Learning Models}
\begin{itemize}
    \item The Random Forest Regressor outperformed Linear Regression and Decision Tree models in predicting the Ad Fatigue Index.
    \item The most important features influencing advertisement fatigue included ad interruption frequency, ad format type, and most annoying ad timing.
\end{itemize}

\subsection{From K-Means Clustering}
\begin{itemize}
    \item Three distinct user segments were identified: High Tolerance, Moderate Fatigue, and High Fatigue users.
    \item High Fatigue users represent a critical at-risk segment most likely to uninstall apps or seek ad-free alternatives.
\end{itemize}

\subsection{From Time-Series Forecasting}
\begin{itemize}
    \item Advertisement complaint volume shows discernible weekly seasonality patterns, with peak complaint periods likely corresponding to high app usage days.
    \item The Prophet model forecasts a continued volume of advertisement-related complaints in the near-term future, reinforcing the need for proactive ad strategy optimization.
\end{itemize}

\section{Business Recommendations}
% ======================================================

Based on the findings of this analysis, the following actionable business recommendations are proposed:

\begin{enumerate}
    \item \textbf{Reduce Interstitial Advertisements}: Full-screen interstitial ads are the most disruptive format. App developers should minimize their frequency, particularly during active user sessions.
    \item \textbf{Increase Rewarded Advertisement Usage}: Rewarded video ads, where users opt in to watch an ad in exchange for in-app rewards, are the most tolerated format. Expanding their share of ad inventory can improve user satisfaction.
    \item \textbf{Implement Ad Frequency Capping}: Limit the number of times the same user sees the same advertisement per session to reduce repetition fatigue.
    \item \textbf{Target the High-Fatigue Segment Differently}: Users in the High Fatigue cluster should be offered freemium or ad-free subscription options rather than continued ad exposure.
    \item \textbf{Optimize Ad Timing}: Avoid displaying advertisements during critical user task moments (e.g., during gameplay, during content loading). The most annoying ad timings identified in the survey should be avoided.
    \item \textbf{Category-Specific Strategies}: App categories with the highest complaint rates (Games, Entertainment) require more careful advertising strategies than lower-complaint categories.
    \item \textbf{Monitor Ad Complaint Trends}: Use the Prophet forecasting model as an ongoing monitoring tool to detect increases in advertisement-related complaints before they impact ratings and uninstall rates.
\end{enumerate}

\section{Conclusion}
% ======================================================

This project presents a comprehensive business analytics study of advertisement fatigue in mobile applications. By combining structured survey data, NLP-based review analysis, machine learning regression, user segmentation, and time-series forecasting, the study delivers a multi-dimensional understanding of how advertisements affect mobile app user experience.

The Ad Fatigue Index provides a quantitative measure of the degree to which users experience frustration from advertisements, and the machine learning models demonstrate that this index can be predicted from demographic and behavioral features with reasonable accuracy. The K-Means clustering reveals three distinct user segments with markedly different levels of advertisement tolerance, which can directly inform differentiated advertising and monetization strategies.

The sentiment analysis of Google Play Store reviews corroborates the survey findings: advertisement complaints are frequent, predominantly negative, and concentrated in specific app categories. The Prophet time-series forecast confirms that advertisement-related dissatisfaction is a persistent and ongoing challenge.

The interactive dashboard developed for this project enables business stakeholders to explore these findings intuitively, supporting data-driven decision-making without requiring technical expertise. Together, the analytical outputs of this project provide actionable guidance for mobile application developers and digital marketers seeking to balance advertising revenue with user satisfaction.

Future work could extend this study by incorporating larger survey samples, applying deep-learning-based sentiment analysis, and integrating real-time data pipelines for continuous monitoring of advertisement complaint trends.
